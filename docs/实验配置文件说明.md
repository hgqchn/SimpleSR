# 实验配置文件说明

本文档说明 SimpleSR 实验配置文件的结构。当前训练入口使用 `OmegaConf` 读取 YAML，支持 `${...}` 插值和命令行 dotlist 覆盖。

训练示例：

```powershell
python train.py -opt configs/example_swinir_x4_train.yaml
```

测试示例：

```powershell
python test.py -opt configs/example_swinir_x4_train.yaml
```

命令行覆盖示例：

```powershell
python train.py -opt configs/example_swinir_x4_train.yaml train.total_iter=1000 log_settings.train_pbar=false
```

## 1. 顶层字段

一个典型配置包含以下顶层字段：

```yaml
exp_name: swinir_x4_example
output_dir: D:\codes\SimpleSR\output
scale: 4
manual_seed: ~

model:
  ...

path:
  ...

datasets:
  ...

network_g:
  ...

train:
  ...

val:
  ...

log_settings:
  ...
```

| 字段 | 是否必需 | 作用 |
| --- | --- | --- |
| `exp_name` | 必需 | 实验名，用于输出目录、日志、wandb run 名称。 |
| `output_dir` | 必需 | 所有实验输出的根目录。 |
| `scale` | 通常必需 | 超分倍率，可被 dataset、network、metric 引用。 |
| `manual_seed` | 可选 | 随机种子。分布式训练时必须显式设置；非分布式时为空会随机生成。 |
| `model` | 必需 | 模型类配置。 |
| `path` | 可选 | 预训练、resume、自定义输出路径等路径配置。 |
| `datasets` | 必需 | 训练、验证或测试数据集配置。 |
| `network_g` | 当前 SRModel 必需 | 生成器网络配置。 |
| `train` | 训练必需 | 训练超参、loss、optimizer、scheduler。 |
| `val` | 验证/测试需要 | 验证频率、保存图像、指标等。 |
| `log_settings` | 训练建议必需 | 日志频率、checkpoint 频率、进度条、wandb。 |

## 2. model

`model` 决定使用哪个模型类。

```yaml
model_type: SwinIRModel
model:
  name: ${model_type}
  path: simplesr.models.base_sr_model.SRModel
```

| 字段 | 作用 |
| --- | --- |
| `model.name` | 模型名称，仅用于日志展示。 |
| `model.path` | 模型类导入路径。 |
| `model.kwargs` | 可选。当前 `build_model(opt)` 会把完整 `opt` 注入模型构造函数，因此通常不需要写。 |

当前 `build_model(opt)` 会执行：

```python
instantiate_from_config(opt['model'], extra_kwargs={'opt': opt})
```

因此模型类一般写成：

```python
class SRModel(BaseModel):
    def __init__(self, opt):
        ...
```

## 3. path

`path` 是可选配置。`parse_options()` 会根据训练/测试模式补充派生路径。

训练模式下自动生成：

```yaml
path:
  experiments_root: <output_dir>/<exp_name>/train_<time>
  weights: <experiments_root>/weights
  checkpoints: <experiments_root>/checkpoints
  visualization: <experiments_root>/visualization
```

测试模式下自动生成：

```yaml
path:
  results_root: <output_dir>/<exp_name>/test_<time>
  visualization: <results_root>/visualization
```

常用手动配置：

```yaml
path:
  pretrain_network_g: D:\codes\SimpleSR\pretrained\SwinIR_x4.pth
  resume_ckpt: D:\codes\SimpleSR\output\exp\train_xxx\checkpoints\5000.state
  experiments_root: D:\codes\SimpleSR\output\custom_exp
  results_root: D:\codes\SimpleSR\output\custom_test
```

| 字段 | 作用 |
| --- | --- |
| `pretrain_network_g` | 加载生成器预训练权重。`SRModel` 固定 `strict=True` 加载。 |
| `resume_ckpt` | 恢复训练 checkpoint。路径应形如 `<exp_root>/checkpoints/<iter>.state`。 |
| `experiments_root` | 自定义训练输出目录。 |
| `results_root` | 自定义测试输出目录。 |

## 4. datasets

`datasets` 可以包含 `train`、`val`、`val_1`、`test`、`test_1` 等。代码会用 `phase.split('_')[0]` 判断阶段。

### 4.1 训练集配置

```yaml
datasets:
  train:
    path: simplesr.data.dataset.PairedImageDataset
    kwargs:
      name: train_set
      dataroot_gt: D:\path\to\train\HR
      dataroot_lq: D:\path\to\train\LR_x4
      phase: train
      scale: ${scale}
      gt_size: 256
      use_augment: true
      io_backend_opt:
        type: disk
    dataset_enlarge_ratio: 1
    batch_size_per_gpu: 8
    num_worker_per_gpu: 4
    prefetch_mode: cpu
    num_prefetch_queue: 1
    pin_memory: true
    persistent_workers: false
```

| 字段 | 作用 |
| --- | --- |
| `path` | Dataset 类导入路径。 |
| `kwargs.name` | 数据集名称，会保存为 `dataset.name`。 |
| `kwargs.dataroot_gt` | GT 图像目录。 |
| `kwargs.dataroot_lq` | LQ/LR 图像目录。 |
| `kwargs.phase` | 阶段，训练为 `train`。 |
| `kwargs.scale` | 超分倍率。若顶层有 `scale`，`parse_options()` 也会写入 dataset 配置。 |
| `kwargs.gt_size` | 训练裁剪 GT patch 尺寸。 |
| `kwargs.use_augment` | 是否启用翻转/旋转增强。 |
| `kwargs.io_backend_opt` | IO 后端，当前常用 `type: disk`。 |
| `dataset_enlarge_ratio` | 虚拟放大数据集长度，减少频繁重启 dataloader。 |
| `batch_size_per_gpu` | 每张 GPU 的 batch size。 |
| `num_worker_per_gpu` | 每张 GPU 的 DataLoader worker 数。 |
| `prefetch_mode` | `cpu` 或空。`cpu` 使用 `PrefetchDataLoader`。 |
| `num_prefetch_queue` | CPU 预取队列长度。 |
| `pin_memory` | 是否启用 DataLoader pin memory。 |
| `persistent_workers` | 是否跨 epoch 保留 worker。 |

### 4.2 验证/测试集配置

```yaml
datasets:
  val:
    path: simplesr.data.dataset.PairedImageDataset
    kwargs:
      name: val_set
      dataroot_gt: D:\path\to\val\HR
      dataroot_lq: D:\path\to\val\LR_x4
      phase: val
      scale: ${scale}
      use_augment: false
      io_backend_opt:
        type: disk
    batch_size: 4
    num_worker: 2
    pin_memory: true
```

验证/测试阶段使用：

| 字段 | 作用 |
| --- | --- |
| `batch_size` | 验证/测试 batch size。默认 1。 |
| `num_worker` | 验证/测试 worker 数。默认 0。 |
| `pin_memory` | 是否启用 pin memory。 |

## 5. network_g

`network_g` 是生成器网络配置，传给 `build_network()`。

```yaml
network_g:
  name: SwinIR
  path: simplesr.networks.SwinIR.SwinIR
  kwargs:
    upscale: ${scale}
    in_chans: 3
    img_size: 64
    window_size: 8
    img_range: 1.0
    depths: [6, 6, 6, 6, 6, 6]
    embed_dim: 180
    num_heads: [6, 6, 6, 6, 6, 6]
    mlp_ratio: 2
    upsampler: pixelshuffle
    resi_connection: 1conv
```

| 字段 | 作用 |
| --- | --- |
| `name` | 网络名称，仅用于日志。 |
| `path` | 网络类导入路径。 |
| `kwargs` | 传给网络构造函数的参数。 |

## 6. train

`train` 只在训练模式使用。

```yaml
train:
  ema_decay: 0.999
  total_iter: 1000000
  warmup_iter: -1

  pixel_opt:
    path: simplesr.losses.basic_loss.L1Loss
    kwargs:
      loss_weight: 1.0
      reduction: mean

  perceptual_opt:
    path: simplesr.losses.basic_loss.PerceptualLoss
    kwargs:
      layer_weights:
        conv5_4: 1.0
      vgg_type: vgg19
      use_input_norm: true
      perceptual_weight: 1.0
      style_weight: 0.0

  optim_g:
    type: Adam
    lr: 2.0e-4
    weight_decay: 0
    betas: [0.9, 0.99]

  scheduler:
    type: MultiStepLR
    milestones: [250000, 400000, 450000, 475000]
    gamma: 0.5
```

| 字段 | 作用 |
| --- | --- |
| `ema_decay` | EMA 衰减率。大于 0 时创建 `net_g_ema`。 |
| `total_iter` | 总训练迭代数。 |
| `warmup_iter` | warm-up 迭代数，`-1` 表示关闭。 |
| `pixel_opt` | 像素损失配置。 |
| `perceptual_opt` | 感知/风格损失配置，可省略。 |
| `optim_g` | 生成器优化器配置。 |
| `scheduler` | 学习率调度器配置。 |

当前 `SRModel` 要求 `pixel_opt` 和 `perceptual_opt` 至少配置一个。

## 7. val

`val` 控制训练过程验证和测试脚本评估。

```yaml
val:
  val_freq: 5000
  save_img: false
  pbar: true
  suffix: ~
  metrics:
    psnr:
      type: calculate_psnr
      crop_border: ${scale}
      test_y_channel: false
      better: higher
    ssim:
      type: calculate_ssim
      crop_border: ${scale}
      test_y_channel: false
      better: higher
```

| 字段 | 作用 |
| --- | --- |
| `val_freq` | 训练时每隔多少 iteration 验证一次。 |
| `save_img` | 是否保存输出图像。 |
| `pbar` | 是否显示验证进度条。 |
| `suffix` | 测试保存图像时的文件名后缀。 |
| `metrics` | 指标配置字典。 |
| `metrics.<name>.type` | 指标函数名，如 `calculate_psnr`、`calculate_ssim`。 |
| `crop_border` | 计算指标前裁剪边界像素数。 |
| `test_y_channel` | 是否只在 Y 通道计算。 |
| `better` | 指标方向，`higher` 或 `lower`，用于最佳指标记录。 |

当前 `SRModel.nondist_validation()` 要求 `val.metrics` 存在。

## 8. log_settings

```yaml
log_settings:
  print_freq: 100
  save_checkpoint_freq: 5000
  csv_flush_freq: 10
  train_pbar: true
  wandb:
    project: simplesr
    resume_id: ~
    mode: disabled
```

| 字段 | 作用 |
| --- | --- |
| `print_freq` | 每隔多少 iteration 打印一次训练日志。 |
| `save_checkpoint_freq` | 每隔多少 iteration 保存一次模型和 checkpoint。 |
| `csv_flush_freq` | 训练 CSV 缓冲多少行后写盘，默认 10。 |
| `train_pbar` | 是否显示训练进度条。只在主进程显示。 |
| `wandb.project` | wandb 项目名。 |
| `wandb.resume_id` | wandb resume id，可为空。 |
| `wandb.mode` | `online`、`offline` 或 `disabled`。 |

debug 模式下：

- 不写配置文件。
- 不写 CSV。
- 不保存 checkpoint。
- 不初始化 wandb。
- 强制关闭验证图像保存。

## 9. 训练输出

训练模式下默认输出目录：

```text
<output_dir>/<exp_name>/train_<time>/
  config.yaml
  log_<time>.log
  train_log.csv
  val_log.csv
  weights/
    net_g_<iter>.pth
    net_g_ema_<iter>.pth
  checkpoints/
    <iter>.state
  visualization/
```

`train_log.csv` 记录训练 loss 和学习率。

`val_log.csv` 记录验证数据集和指标。

## 10. 测试输出

测试模式下默认输出目录：

```text
<output_dir>/<exp_name>/test_<time>/
  config.yaml
  log_<time>.log
  test_log.csv
  visualization/
```

测试脚本会跳过 `datasets.train`，只使用 `val/test` 阶段的数据集。
