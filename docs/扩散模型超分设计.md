# 扩散模型超分设计

本文档说明 SimpleSR 中扩散模型超分的当前实现方式。基础训练框架仍复用 `simplesr/models/` 下的 `BaseModel` / `SRModel`，DDPM 相关训练壳和扩散原理放在 `diffusion/ddpm/` 下。

## 1. 总体思路

保持设计简单：

```text
simplesr/
  models/
    base_model.py
    base_sr_model.py            # 确定性 SR 训练壳，可被扩散 SR 复用验证/保存逻辑

  diffusion/
    common_utils.py             # timestep embedding、schedule 抽取、mean_flat 等通用工具
    ddpm/
      diffusion_sr_model.py     # DDPM 超分训练壳，继承 SRModel
      gaussian_diffusion.py     # DDPM/DDIM 和 GaussianDiffusionSR
      respace.py                # 类 IDDPM 的少步采样/跳步扩散
      resample.py               # timestep sampler
      script_util.py            # DDPM-SR 模型和 diffusion 构建工具
      unet.py                   # UNetModel / SuperResModel
```

职责边界：

| 模块 | 职责 |
| --- | --- |
| `diffusion/ddpm/diffusion_sr_model.py::DiffusionSRModel` | 继承 `SRModel`，只覆盖 DDPM 训练 loss 和采样；网络构建、验证、保存尽量复用 `SRModel`。 |
| `diffusion/ddpm/*.py` | DDPM/DDIM、少步采样、timestep sampler、UNet 等具体扩散超分实现。 |
| 普通扩散类 | 不带超分条件，描述原始生成过程。 |
| 超分扩散类 | 继承同文件中的普通扩散类，主要增加 LR 条件输入和条件注入方式。 |

推荐层级：

```text
BaseModel
  └── DiffusionSRModel
        ├── net_g: denoiser / score model / velocity model
        ├── diffusion: 训练扩散过程，例如 GaussianDiffusionSR(1000 steps)
        └── sample_diffusion: 采样扩散过程，例如 SpacedDiffusionSR(ddim10)
```

`DiffusionSRModel` 不需要知道底层是 DDPM 还是 Flow Matching，只要求训练和采样 SR 扩散类提供统一接口：

```python
training_losses(model, gt, lq)
sample(model, lq, shape)
```

## 2. 文件职责

### 2.1 `base_diffusion.py`

放公共基类和跨扩散方法可复用的小工具。

建议包含：

```python
class BaseDiffusion:
    def training_losses(self, model, x_start, *args, **kwargs):
        raise NotImplementedError

    @torch.no_grad()
    def sample(self, model, shape, *args, **kwargs):
        raise NotImplementedError
```

如果不想引入强基类，也可以只放工具函数和轻量 mixin。但建议至少定义统一接口，方便 `DiffusionSRModel` 调用。

适合放在这里的内容：

| 内容 | 说明 |
| --- | --- |
| `extract_into_tensor()` | 从一维 schedule 中按 timestep 取值并 reshape 到 batch。 |
| `mean_flat()` | 对非 batch 维度求平均，用于 loss。 |
| `normal_kl()` | KL 计算，如果实现 VLB loss 会用到。 |
| `discretized_gaussian_log_likelihood()` | DDPM 类似实现可能用到。 |
| `BaseDiffusion` | 普通扩散统一接口。 |
| `BaseDiffusionSR` | 可选，SR 扩散统一接口。 |

### 2.2 `schedules.py`

放 schedule 构造函数。

建议包含：

```python
def make_beta_schedule(schedule, n_timestep):
    ...

def betas_for_alpha_bar(num_diffusion_timesteps, alpha_bar, max_beta=0.999):
    ...
```

常见 schedule：

| 名称 | 说明 |
| --- | --- |
| `linear` | DDPM 原始线性 beta。 |
| `cosine` | Improved DDPM 常用 cosine schedule。 |
| `sqrt` | 可选，部分 SR 扩散会用。 |

### 2.3 `respace.py`

放 timestep 重采样工具，服务于少步采样。

建议包含：

```python
def space_timesteps(num_timesteps, section_counts):
    ...
```

作用类似 IDDPM 中的 timestep respacing：

```yaml
diffusion:
  timesteps: 1000
  timestep_respacing: ddim50
```

常见形式：

| 配置 | 含义 |
| --- | --- |
| `ddim50` | 从 1000 步中选 50 个 timestep，用 DDIM 风格采样。 |
| `[10, 15, 20]` | 分段选择 timestep。 |
| `50` | 均匀选 50 步。 |

### 2.4 `gaussian_diffusion.py`

放 DDPM/DDIM 类型扩散。

建议包含两个类：

```python
class GaussianDiffusion(BaseDiffusion):
    ...

class GaussianDiffusionSR(GaussianDiffusion):
    ...
```

`GaussianDiffusion` 负责原始扩散逻辑，`GaussianDiffusionSR` 继承它并增加低分图像条件注入。

### 2.5 `spaced_diffusion.py`

放类似 IDDPM 的少步采样/跳步扩散封装。

建议包含：

```python
class SpacedDiffusion(GaussianDiffusion):
    ...

class SpacedDiffusionSR(GaussianDiffusionSR):
    ...
```

它的职责不是重新定义扩散原理，而是基于 `use_timesteps` 把原始 1000 步 diffusion 映射到较少步数。

核心功能：

| 功能 | 说明 |
| --- | --- |
| 保存 `use_timesteps` | 实际使用的 timestep 子集。 |
| 构造新 beta | 根据选中的 timestep 重新计算等效 beta。 |
| 包装模型 timestep | 将少步索引映射回原始 timestep。 |
| 支持 DDIM/少步采样 | 验证和测试时减少采样成本。 |

### 2.6 `flow_matching.py`

放 Flow Matching 或 Rectified Flow 类型方法。

建议包含：

```python
class FlowMatching(BaseDiffusion):
    ...

class FlowMatchingSR(FlowMatching):
    ...
```

Flow Matching 与 DDPM 的主要区别：

| 项 | DDPM 类型 | Flow Matching 类型 |
| --- | --- | --- |
| 训练目标 | 预测 noise / x0 / v | 预测 velocity / flow |
| 前向过程 | 离散加噪 timestep | 连续时间插值路径 |
| 采样过程 | 反向扩散迭代 | ODE 积分或 Euler 步进 |
| SR 条件 | 给 denoiser 注入 LR | 给 velocity model 注入 LR |

`DiffusionSRModel` 不应关心这些差异，只调用统一的 `training_losses()` 和 `sample()`。

## 3. DDPM 类型扩散接口

DDPM 类型的普通扩散类建议提供以下接口。

### 3.1 初始化

```python
class GaussianDiffusion(BaseDiffusion):
    def __init__(
        self,
        betas,
        model_mean_type='eps',
        model_var_type='fixed_small',
        loss_type='mse',
        rescale_timesteps=False,
    ):
        ...
```

建议初始化时预计算：

| 缓存变量 | 作用 |
| --- | --- |
| `betas` | beta schedule。 |
| `alphas` | `1 - betas`。 |
| `alphas_cumprod` | 累积 alpha。 |
| `alphas_cumprod_prev` | 上一个 timestep 的累积 alpha。 |
| `sqrt_alphas_cumprod` | q_sample 计算用。 |
| `sqrt_one_minus_alphas_cumprod` | q_sample 计算用。 |
| `posterior_mean_coef1/2` | q posterior mean 计算用。 |
| `posterior_variance` | q posterior variance。 |
| `posterior_log_variance_clipped` | 防止 log 0。 |

### 3.2 前向加噪

```python
def q_sample(self, x_start, t, noise=None):
    """从 q(x_t | x_0) 采样。"""
```

输入输出：

| 参数 | 说明 |
| --- | --- |
| `x_start` | 原始清晰图像，SR 中通常是 GT。 |
| `t` | timestep，shape 通常为 `(B,)`。 |
| `noise` | 可选噪声，不传则随机生成。 |
| 返回 | `x_t`。 |

### 3.3 q posterior

```python
def q_posterior_mean_variance(self, x_start, x_t, t):
    """计算 q(x_{t-1} | x_t, x_0)。"""
```

返回：

```python
posterior_mean, posterior_variance, posterior_log_variance
```

### 3.4 模型预测封装

```python
def model_predictions(self, model, x, t, clip_denoised=True, denoised_fn=None, model_kwargs=None):
    """调用模型并统一转换为 pred_noise / pred_xstart。"""
```

当前保留两类目标：

| 目标 | 说明 |
| --- | --- |
| `eps` / `pred_noise` | 模型预测噪声。 |
| `x_start` / `pred_x0` | 模型预测干净图像。 |

该函数不是 ADM 原版接口，但能让 `p_mean_variance()` 和 `ddim_sample()` 更易读；不新增采样能力。

### 3.5 反向一步

```python
def p_mean_variance(self, model, x, t, clip_denoised=True, denoised_fn=None, model_kwargs=None):
    """计算 p(x_{t-1} | x_t) 的 mean/variance。"""
```

返回建议用 dict：

```python
{
    "mean": model_mean,
    "variance": model_variance,
    "log_variance": model_log_variance,
    "pred_xstart": pred_xstart,
}
```

### 3.6 DDPM 采样一步

```python
def p_sample(self, model, x, t, clip_denoised=True, denoised_fn=None, cond_fn=None, model_kwargs=None):
    """从 p(x_{t-1} | x_t) 采样一步。"""
```

返回：

```python
{
    "sample": x_prev,
    "pred_xstart": pred_xstart,
}
```

### 3.7 DDPM 完整采样

```python
def p_sample_loop(self, model, shape, noise=None, clip_denoised=True, denoised_fn=None, cond_fn=None, model_kwargs=None, device=None, progress=False):
    """从纯噪声开始迭代采样。"""
```

### 3.8 DDIM 采样

```python
@torch.no_grad()
def ddim_sample(self, model, x, t, clip_denoised=True, denoised_fn=None, cond_fn=None, model_kwargs=None, eta=0.0):
    ...

@torch.no_grad()
def ddim_sample_loop(self, model, shape, noise=None, clip_denoised=True, denoised_fn=None, cond_fn=None, model_kwargs=None, device=None, progress=False, eta=0.0):
    ...
```

`eta=0` 时为确定性 DDIM。

### 3.9 训练 loss

```python
def training_losses(self, model, x_start, t, model_kwargs=None, noise=None):
    """计算普通扩散训练 loss。"""
```

建议行为：

1. 上层 sampler 先采样 timestep，并传入 `t`。
2. 如果 `noise is None`，随机生成噪声。
3. 调用 `q_sample(x_start, t, noise)` 得到 `x_t`。
4. 调用模型预测。
5. 根据目标类型计算 MSE。
6. 返回 ADM 风格的 `terms` 字典。

返回：

```python
{
    "loss": mse,
    "mse": mse.detach(),
}
```

## 4. DDPM 超分扩散接口

`GaussianDiffusionSR` 继承 `GaussianDiffusion`，主要改变模型输入和采样接口。

建议统一接口：

```python
class GaussianDiffusionSR(GaussianDiffusion):
    def training_losses(self, model, gt, lq, t=None, noise=None):
        ...

    @torch.no_grad()
    def sample(self, model, lq, shape=None, noise=None):
        ...
```

### 4.1 条件注入

第一版可以直接使用 bicubic 上采样后的 LR 作为条件：

```python
def prepare_sr_condition(self, lq, gt_shape):
    lq_up = F.interpolate(lq, size=gt_shape[-2:], mode='bicubic', align_corners=False)
    return lq_up
```

模型输入可以在 SR 类中统一构造：

```python
model_input = torch.cat([x_t, lq_up], dim=1)
model_output = model(model_input, t)
```

这样 `DiffusionSRModel` 不需要知道条件是拼接、cross attention 还是 residual，只把 `gt/lq` 交给 diffusion。

### 4.2 训练 loss

```python
def training_losses(self, model, gt, lq, t=None, noise=None):
    if t is None:
        t = torch.randint(0, self.num_timesteps, (gt.size(0),), device=gt.device)
    if noise is None:
        noise = torch.randn_like(gt)

    x_t = self.q_sample(gt, t, noise)
    lq_cond = self.prepare_sr_condition(lq, gt.shape)
    model_input = self.build_model_input(x_t, lq_cond)
    model_output = model(model_input, t)
    loss = self.compute_loss(model_output, gt, noise, x_t, t)
    return loss, {"loss": loss.detach()}
```

### 4.3 采样

```python
@torch.no_grad()
def sample(self, model, lq, shape=None, noise=None):
    if shape is None:
        shape = (
            lq.size(0),
            3,
            lq.size(2) * self.scale,
            lq.size(3) * self.scale,
        )
    lq_cond = self.prepare_sr_condition(lq, shape)
    return self.p_sample_loop(
        model,
        shape=shape,
        noise=noise,
        lq_cond=lq_cond,
    )
```

`p_mean_variance()`、`p_sample()`、`ddim_sample()` 等函数需要把 `lq_cond` 通过 `model_kwargs` 一路传给模型输入构造逻辑。

## 5. SpacedDiffusion / IDDPM 少步采样

`spaced_diffusion.py` 负责少步采样，不建议把这部分混在 `GaussianDiffusion` 主类里。

建议类：

```python
class SpacedDiffusion(GaussianDiffusion):
    def __init__(self, use_timesteps, betas, **kwargs):
        ...

class SpacedDiffusionSR(GaussianDiffusionSR):
    def __init__(self, use_timesteps, betas, **kwargs):
        ...
```

核心接口：

```python
def _WrappedModel(model, timestep_map, rescale_timesteps):
    ...
```

核心逻辑：

| 步骤 | 说明 |
| --- | --- |
| 输入原始 `betas` 和 `use_timesteps` | 例如原始 1000 步中选 50 步。 |
| 重新构造等效 beta | 保证少步过程的 alpha cumprod 与原始选中步一致。 |
| 保存 `timestep_map` | 将少步索引映射回原始 timestep。 |
| 包装模型调用 | 模型仍可接收原始 timestep 或 rescale 后 timestep。 |

配置示例：

```yaml
diffusion:
  type: spaced_gaussian_sr
  timesteps: 1000
  beta_schedule: cosine
  timestep_respacing: ddim50
  objective: pred_noise
```

## 6. Flow Matching 接口

Flow Matching 可以放在独立文件，不强行套 DDPM 函数名。

```python
class FlowMatching(BaseDiffusion):
    def training_losses(self, model, x_start, t=None, noise=None, **model_kwargs):
        ...

    @torch.no_grad()
    def sample(self, model, shape, noise=None, steps=50, **model_kwargs):
        ...

class FlowMatchingSR(FlowMatching):
    def training_losses(self, model, gt, lq, t=None, noise=None):
        ...

    @torch.no_grad()
    def sample(self, model, lq, shape=None, noise=None, steps=50):
        ...
```

和 DDPM 的差异：

| 项 | DDPM 类型 | Flow Matching 类型 |
| --- | --- | --- |
| 训练目标 | noise / x0 / v | velocity / flow |
| 时间变量 | 离散 timestep | 连续时间 `t in [0, 1]` |
| 前向过程 | q_sample 加噪 | x0 到 x1 的插值路径 |
| 采样过程 | 反向扩散 | ODE/Euler/Heun 积分 |
| SR 条件 | 注入 LR 到 denoiser | 注入 LR 到 velocity model |

`DiffusionSRModel` 仍然只调用：

```python
loss, log_dict = self.diffusion.training_losses(self.net_g, self.gt, self.lq)
self.output = self.diffusion.sample(self.net_g, self.lq)
```

## 7. DiffusionSRModel 训练壳

`diffusion/ddpm/diffusion_sr_model.py` 只负责 DDPM 超分的训练和测试流程。

建议核心逻辑：

```python
class DiffusionSRModel(SRModel):
    def __init__(self, opt):
        self.diffusion = instantiate_from_config(opt["diffusion"]["train"])
        self.sample_diffusion = instantiate_from_config(opt["diffusion"]["val"])
        super().__init__(opt)

    def feed_data(self, data):
        self.lq = data['lq'].to(self.device)
        self.gt = data['gt'].to(self.device)

    def optimize_parameters(self, current_iter):
        self.optimizer_g.zero_grad()
        loss, loss_dict = ddpm_training_step(
            self.net_g,
            self.diffusion,
            self.gt,
            lq=self.lq,
            schedule_sampler=self.schedule_sampler,
        )
        loss.backward()
        self.optimizer_g.step()
        self.log_dict = self.reduce_loss_dict(loss_dict)

        if self.ema_decay > 0:
            self.update_ema(self.net_g, self.net_g_ema)

    @torch.no_grad()
    def test(self):
        net = self.get_ema_module(self.net_g_ema) if hasattr(self, 'net_g_ema') else self.net_g
        net.eval()
        self.output = self.sample_diffusion.sample(net, self.lq, sampler=self.diffusion_val_opt.get("sampler", "ddpm"))
```

当前实现直接继承 `SRModel`，因此 `feed_data()`、`get_current_visuals()`、`nondist_validation()`、`dist_validation()`、`save()` 都复用基础超分模型逻辑。

## 8. diffusion 配置建议

DDPM SR 推荐配置：训练使用完整 1000 步 `GaussianDiffusionSR`，验证/测试使用少步 `SpacedDiffusionSR` + DDIM。

```yaml
diffusion:
  train:
    name: GaussianDiffusionSR
    path: diffusion.ddpm.gaussian_diffusion.GaussianDiffusionSR
    schedule_sampler: uniform
    kwargs:
      diffusion_steps: 1000
      noise_schedule: linear
      objective: pred_noise
      scale: ${scale}
  val:
    name: SpacedDiffusionSR
    path: diffusion.ddpm.respace.SpacedDiffusionSR
    sampler: ddim
    kwargs:
      diffusion_steps: 1000
      noise_schedule: linear
      timestep_respacing: ddim50
      objective: pred_noise
      scale: ${scale}
```

Flow Matching SR：

```yaml
diffusion:
  type: flow_matching_sr
  path: simplesr.diffusion.flow_matching.FlowMatchingSR
  kwargs:
    steps: 50
    objective: velocity
    scale: ${scale}
    condition_mode: concat
```

## 9. 建议实现顺序

1. 先在 `diffusion/ddpm/gaussian_diffusion.py` 中保持 `training_losses()` / `sample()` 统一接口。
2. 在 `diffusion/ddpm/unet.py` 中让 `SuperResModel.forward(x_t, t, lq=...)` 接收低分条件。
3. 在 `diffusion/ddpm/diffusion_sr_model.py` 中继承 `SRModel`，只覆盖 `init_training_settings()`、`optimize_parameters()`、`test()`。
4. 在配置中将 `model.path` 指向 `diffusion.ddpm.diffusion_sr_model.DiffusionSRModel`。
5. 需要更复杂采样时优先扩展 `respace.py` / `GaussianDiffusionSR.sample()`，不要另写训练循环。
