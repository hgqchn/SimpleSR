# 基础模型和基础超分模型

本文档说明 `simplesr/models/base_model.py` 和 `simplesr/models/base_sr_model.py` 的职责边界、主要成员、子类实现约定，以及后续扩展扩散超分模型时的建议。

## 1. 模型层级

当前模型层级：

```text
BaseModel
  └── SRModel
```

- `BaseModel`：训练基础设施基类，负责设备选择、DDP 包装、优化器、学习率调度、EMA、权重加载、checkpoint 保存恢复、loss 汇总和验证分发入口。
- `SRModel`：确定性单图像超分模型，负责构建生成器 `net_g`、构建 loss、执行训练迭代、推理、验证、保存超分结果图像和指标。

`BaseModel` 不包含具体任务逻辑。超分、扩散、GAN 等任务应由子类实现。

## 2. BaseModel 成员

| 成员 | 来源 | 作用 |
| --- | --- | --- |
| `self.opt` | `__init__(opt)` | 完整实验配置。 |
| `self.local_rank` | `get_local_rank()` | 当前进程本地 rank。非分布式时通常为 0。 |
| `self.device` | 自动判断 | CUDA 可用时使用 `cuda:{local_rank}`，否则使用 CPU。 |
| `self.is_train` | `opt['is_train']` | 当前是否为训练模式。 |
| `self.schedulers` | 空列表 | 保存所有 scheduler，用于统一 step、保存和恢复。 |
| `self.optimizers` | 空列表 | 保存所有 optimizer，用于保存、恢复和查询学习率。 |

子类通常还会创建以下成员：

| 成员 | 建议创建位置 | 作用 |
| --- | --- | --- |
| `self.net_g` | 子类 `__init__` | 主网络。当前通用 checkpoint 默认保存该网络。 |
| `self.net_g_ema` | 子类训练初始化 | EMA 网络，仅在启用 EMA 时创建。 |
| `self.optimizer_g` | 子类 `setup_optimizers()` | 主网络优化器。 |
| `self.log_dict` | 子类 `optimize_parameters()` | 当前迭代日志，如 loss。 |
| `self.lq` | 子类 `feed_data()` | 低质量输入。 |
| `self.gt` | 子类 `feed_data()` | GT 图像。 |
| `self.output` | 子类训练或推理 | 模型输出。 |

## 3. BaseModel 函数

### 3.1 子类需要实现的接口

| 函数 | 是否必须 | 作用 |
| --- | --- | --- |
| `feed_data(self, data)` | 必须 | 从 dataloader batch 中取数据并移动到 `self.device`。 |
| `optimize_parameters(self, current_iter)` | 训练模型必须 | 执行一次训练迭代，包括前向、loss、反向、optimizer step 和日志更新。 |
| `get_current_visuals(self)` | 验证/测试模型必须 | 返回当前可视化结果，如 `lq/result/gt`。 |
| `save(self, epoch, current_iter)` | 训练模型必须 | 保存网络权重和训练状态。 |
| `nondist_validation(...)` | 需要验证时必须 | 非分布式验证逻辑。 |
| `dist_validation(...)` | 分布式验证时必须 | 分布式验证逻辑，通常只在 rank 0 执行真正验证。 |

### 3.2 验证相关

| 函数 | 作用 |
| --- | --- |
| `validation(...)` | 根据 `opt['distributed']` 分发到 `dist_validation()` 或 `nondist_validation()`，并返回验证结果。 |
| `_initialize_best_metric_results(dataset_name)` | 初始化某个验证集的最佳指标记录。 |
| `_update_best_metric_result(dataset_name, metric, val, current_iter)` | 根据 `better: higher/lower` 更新最佳指标。 |

### 3.3 EMA 相关

| 函数 | 作用 |
| --- | --- |
| `init_ema(network, decay=0.999)` | 基于已有网络创建 `ModelEmaV2`，关闭梯度并设为 eval。 |
| `update_ema(network, net_ema)` | 用当前网络参数更新 EMA 权重。 |
| `get_ema_module(net_ema)` | 取出 `ModelEmaV2.module`。 |

### 3.4 设备与分布式

| 函数 | 作用 |
| --- | --- |
| `model_to_device(net)` | 将模型移动到 CUDA 或 CPU；分布式时包装为 `DistributedDataParallel`。CPU 情况下不会传 CUDA 专用的 `device_ids`。 |
| `get_bare_model(net)` | 如果模型被 DDP 包装，返回内部 `.module`。 |
| `print_network(net)` | 仅主进程打印网络类型和参数量。 |

### 3.5 优化器和学习率

| 函数 | 作用 |
| --- | --- |
| `get_optimizer(optim_type, params, lr, **kwargs)` | 根据字符串创建 PyTorch optimizer。 |
| `setup_schedulers()` | 根据 `opt['train']['scheduler']` 为 `self.optimizers` 创建 scheduler。 |
| `_set_lr(lr_groups_l)` | 手动设置参数组学习率，用于 warm-up。 |
| `_get_init_lr()` | 获取 optimizer 参数组中的初始学习率。 |
| `update_learning_rate(current_iter, warmup_iter=-1)` | 每个 iteration 更新 scheduler，并在 warm-up 阶段线性调整 lr。 |
| `get_current_learning_rate()` | 返回第一个 optimizer 的所有参数组 lr。 |

注意：当前 `setup_schedulers()` 会对配置中的 `scheduler` 执行 `pop('type')`，会修改原始配置。后续如果需要重复初始化，建议改为先复制配置。

### 3.6 保存与恢复

| 函数 | 作用 |
| --- | --- |
| `save_network(net, net_label, current_iter)` | 保存单个网络权重到 `opt['path']['weights']`。 |
| `load_network(net, load_path, strict=True)` | 从权重文件加载网络参数，当前 `torch.load` 显式使用 `weights_only=False`。 |
| `_print_different_keys_loading(...)` | 加载前打印缺失、多余或尺寸不一致的 key。 |
| `save_checkpoint(epoch, current_iter)` | 保存训练恢复所需状态到 `opt['path']['checkpoints']`，返回保存路径。 |
| `resume_checkpoint(resume_state)` | 恢复 `net_g`、`net_g_ema`、optimizers 和 schedulers。 |

当前 checkpoint 结构：

```python
{
    "epoch": epoch,
    "iter": current_iter,
    "net_g": self.get_bare_model(self.net_g).state_dict(),
    "net_g_ema": net_g_ema.module.state_dict() if net_g_ema else None,
    "optimizers": [...],
    "schedulers": [...],
}
```

### 3.7 loss 汇总

| 函数 | 作用 |
| --- | --- |
| `reduce_loss_dict(loss_dict)` | 分布式训练时对 loss 做 reduce 平均，返回 Python float 日志字典。 |

子类训练时通常这样使用：

```python
loss_dict = OrderedDict()
loss_dict['l_pix'] = l_pix
self.log_dict = self.reduce_loss_dict(loss_dict)
```

## 4. SRModel 说明

`SRModel` 是当前确定性单图像超分实现，适用于 SwinIR 等输入 LR、输出 SR 的模型。

### 4.1 SRModel 需要的 opt 字段

`SRModel.__init__(opt)` 接收完整实验配置，主要依赖：

| 字段 | 作用 |
| --- | --- |
| `is_train` | 是否为训练模式。 |
| `distributed` / `rank` / `world_size` | 分布式训练和验证控制。 |
| `exp_name` | 日志和测试图像命名。 |
| `network_g` | 生成器网络配置，传给 `build_network()`。 |
| `path.weights` | 保存网络权重。 |
| `path.checkpoints` | 保存训练 checkpoint。 |
| `path.visualization` | 保存验证或测试图像。 |
| `path.pretrain_network_g` | 可选，加载预训练生成器权重。 |
| `train.pixel_opt` | 可选，像素损失配置。 |
| `train.perceptual_opt` | 可选，感知损失配置。 |
| `train.optim_g` | 生成器优化器配置。 |
| `train.scheduler` | 学习率调度器配置。 |
| `train.ema_decay` | EMA 衰减率，大于 0 时启用 EMA。 |
| `val.metrics` | 验证指标配置。 |
| `val.save_img` | 是否保存验证图像。 |
| `val.pbar` | 是否显示验证进度条。 |
| `val.suffix` | 测试保存图像时的可选后缀。 |

预训练权重加载固定使用 `strict=True`，没有单独的 `strict_load_g` 配置。

### 4.2 SRModel 成员

| 成员 | 作用 |
| --- | --- |
| `self.net_g` | 生成器网络。 |
| `self.load_path` | 预训练权重路径。 |
| `self.net_g_ema` | EMA 生成器，仅当 `ema_decay > 0` 时创建。 |
| `self.cri_pix` | 像素损失。 |
| `self.cri_perceptual` | 感知/风格损失。 |
| `self.optimizer_g` | 生成器优化器。 |
| `self.lq` | 当前 batch 的低质量输入。 |
| `self.gt` | 当前 batch 的 GT。 |
| `self.output` | 当前模型输出。 |
| `self.metric_results` | 验证指标累计结果。 |

### 4.3 构建与训练函数

| 函数 | 作用 |
| --- | --- |
| `__init__(opt)` | 构建 `net_g`，移动到设备，打印网络，加载预训练权重，训练时初始化训练组件。 |
| `init_training_settings()` | 设置训练模式，初始化 EMA、loss、optimizer 和 scheduler。 |
| `init_losses()` | 根据 `pixel_opt` / `perceptual_opt` 构建 loss；至少需要一个 loss。 |
| `setup_optimizers()` | 收集 `net_g` 中可训练参数，创建 `optimizer_g` 并加入 `self.optimizers`。 |
| `feed_data(data)` | 读取 `data['lq']` 和可选 `data['gt']` 到设备。 |
| `optimize_parameters(current_iter)` | 执行一次训练迭代并更新 `self.log_dict`。 |

### 4.4 推理与验证函数

| 函数 | 作用 |
| --- | --- |
| `test()` | 优先使用 EMA 网络推理，否则使用 `net_g`。 |
| `test_selfensemble()` | 8 路自集成推理，支持 batch 大于 1。 |
| `dist_validation(...)` | 分布式验证入口，仅 rank 0 执行验证并返回结果。 |
| `nondist_validation(...)` | batch 方式遍历验证集，保存图像、计算指标、更新最佳指标，并返回结果字典。 |
| `_log_validation_metric_values(...)` | 输出验证指标日志，并可写入 wandb。 |
| `get_current_visuals()` | 返回 CPU tensor 格式的 `lq/result/gt`。 |

验证集名称来自 `dataloader.dataset.name`，该成员由 dataset 构造参数 `kwargs.name` 设置。

### 4.5 保存函数

| 函数 | 作用 |
| --- | --- |
| `save(epoch, current_iter)` | 保存 `net_g`，如果存在 EMA 则保存 `net_g_ema`，然后保存 checkpoint。 |

## 5. 训练和测试入口的配合

### 5.1 build_model

`simplesr/models/__init__.py` 中的 `build_model(opt)` 接收完整实验配置：

```python
model_opt = opt['model']
model = instantiate_from_config(model_opt, extra_kwargs={'opt': opt})
```

因此模型类构造函数应支持：

```python
def __init__(self, opt):
    ...
```

### 5.2 dataset name

数据集名称放在 dataset 的 `kwargs.name` 中：

```yaml
datasets:
  train:
    path: simplesr.data.dataset.PairedImageDataset
    kwargs:
      name: train_set
```

`PairedImageDataset` 会保存为：

```python
self.name = name
```

训练、验证、测试日志都从 `dataset.name` 读取名称。

### 5.3 CSV、wandb 和进度条

训练入口当前支持：

| 功能 | 配置/行为 |
| --- | --- |
| 本地训练 CSV | 非 debug 模式写入 `train_log.csv`。 |
| 本地验证 CSV | 非 debug 模式写入 `val_log.csv`。 |
| wandb | `log_settings.wandb.mode != disabled` 且非 debug 时启用。 |
| 训练进度条 | `log_settings.train_pbar` 控制，只在主进程显示。 |
| debug 模式 | 不保存配置、CSV、checkpoint，并强制关闭验证图像保存。 |

## 6. 子类最小实现契约

新增一个训练模型类时，建议至少实现：

```python
class MyModel(BaseModel):
    def __init__(self, opt):
        super().__init__(opt)
        # build network(s), load pretrained weights
        # if self.is_train: init losses / optimizers / schedulers

    def feed_data(self, data):
        ...

    def optimize_parameters(self, current_iter):
        ...

    def test(self):
        ...

    def get_current_visuals(self):
        ...

    def nondist_validation(self, dataloader, current_iter, wandb_logger, save_img):
        ...

    def save(self, epoch, current_iter):
        ...
```

训练模型还应保证：

- 至少一个 optimizer 加入 `self.optimizers`。
- 使用 scheduler 时加入 `self.schedulers`。
- `optimize_parameters()` 后 `self.log_dict` 可被 `get_current_log()` 读取。
- 如果想复用当前 checkpoint，主网络建议命名为 `self.net_g`。
