# SimpleSR 模型基类与扩散超分设计说明

本文档整理 `simplesr/models/base_model.py` 与 `simplesr/models/base_sr_model.py` 的职责边界，并给出后续增加扩散模型超分实现时的设计建议。

## 1. 当前模型层级

当前模型层级可以理解为两层：

```text
BaseModel
  └── SRModel
```

- `BaseModel`：训练基础设施基类，负责设备、DDP、优化器、学习率、EMA、保存加载、日志 loss 汇总、验证入口等通用逻辑。
- `SRModel`：单图像超分任务模型，负责构建 `net_g`、定义 loss、执行前向与反向、验证、保存结果图像和指标。

`BaseModel` 不知道具体任务，也不应该直接写超分、扩散、GAN 等任务逻辑；这些应由子类实现。

## 2. BaseModel 成员说明

### 初始化成员

| 成员 | 来源 | 作用 |
| --- | --- | --- |
| `self.opt` | `__init__(opt)` | 全局配置字典，包含训练、验证、路径、分布式等配置。 |
| `self.local_rank` | `get_local_rank()` | 当前进程绑定的本地 GPU 编号。 |
| `self.device` | `cuda:{local_rank}` | 当前模型和数据使用的设备。 |
| `self.is_train` | `opt['is_train']` | 当前是否为训练模式。 |
| `self.schedulers` | 空列表 | 保存所有学习率调度器，供统一 step、保存和恢复。 |
| `self.optimizers` | 空列表 | 保存所有优化器，供统一保存、恢复和获取学习率。 |

### 子类运行时常见成员

这些成员不在 `BaseModel.__init__` 中创建，但当前 `SRModel` 或后续子类通常会使用：

| 成员 | 建议创建位置 | 作用 |
| --- | --- | --- |
| `self.net_g` | 子类 `__init__` | 主生成/恢复网络。当前 checkpoint 保存逻辑依赖该成员。 |
| `self.net_g_ema` | 子类训练初始化 | EMA 包装后的生成器，用于验证或更稳定的推理。 |
| `self.optimizer_g` | 子类 `setup_optimizers` | 生成器优化器。 |
| `self.log_dict` | 子类 `optimize_parameters` | 当前 iteration 的 loss/指标日志字典。 |
| `self.lq` | 子类 `feed_data` | 低质量输入图像。 |
| `self.gt` | 子类 `feed_data` | 高质量 GT 图像，训练和全参考验证时使用。 |
| `self.output` | 子类推理/训练 | 模型输出图像。 |

## 3. BaseModel 函数说明

### 抽象接口

这些函数在 `BaseModel` 中只是占位，具体子类应实现。

| 函数 | 子类是否必须实现 | 作用 |
| --- | --- | --- |
| `feed_data(self, data)` | 必须 | 从 dataloader batch 中取数据并移动到 `self.device`。 |
| `optimize_parameters(self, current_iter)` | 训练模型必须 | 执行一次训练迭代：前向、loss、反向、optimizer step、日志记录。 |
| `get_current_visuals(self)` | 验证/测试模型必须 | 返回当前可视化结果，如 `lq/result/gt`。 |
| `save(self, epoch, current_iter)` | 训练模型必须 | 保存网络权重与 checkpoint。 |

### 验证相关

| 函数 | 作用 |
| --- | --- |
| `validation(...)` | 根据 `opt['dist']` 分发到 `dist_validation` 或 `nondist_validation`。 |
| `_initialize_best_metric_results(dataset_name)` | 初始化每个验证集的最佳指标记录。 |
| `_update_best_metric_result(dataset_name, metric, val, current_iter)` | 根据 higher/lower 规则更新最佳指标。 |

注意：`dist_validation` 和 `nondist_validation` 不是 `BaseModel` 中实现的通用逻辑，当前由 `SRModel` 提供。新的任务子类如果需要验证，应实现这两个函数或至少实现 `nondist_validation`。

### EMA 相关

| 函数 | 作用 |
| --- | --- |
| `init_ema(network, decay=0.999)` | 基于已有网络创建 `ModelEmaV2` 包装对象，关闭梯度并设为 eval。 |
| `update_ema(network, net_ema)` | 用当前网络参数更新 EMA 权重。 |
| `get_ema_module(net_ema)` | 取出 `ModelEmaV2.module`，用于推理或保存。 |

EMA 适合确定性 SR、GAN 生成器、扩散模型的 denoiser/UNet。扩散模型训练时通常对 denoiser 使用 EMA。

### 设备与分布式

| 函数 | 作用 |
| --- | --- |
| `model_to_device(net)` | 把模型移动到当前 GPU；分布式训练时包成 `DistributedDataParallel`。 |
| `get_bare_model(net)` | 如果模型被 DDP 包装，返回内部 `.module`。 |
| `print_network(net)` | 仅主进程打印模型结构和参数量。 |

### 优化器与学习率

| 函数 | 作用 |
| --- | --- |
| `get_optimizer(optim_type, params, lr, **kwargs)` | 根据字符串创建 PyTorch 优化器。 |
| `setup_schedulers()` | 根据 `opt['train']['scheduler']` 给 `self.optimizers` 中的优化器创建 scheduler。 |
| `_set_lr(lr_groups_l)` | 手动写回学习率，主要用于 warm-up。 |
| `_get_init_lr()` | 读取 optimizer 参数组中的 `initial_lr`。 |
| `update_learning_rate(current_iter, warmup_iter=-1)` | 每次迭代更新 scheduler，并在 warm-up 阶段线性调整 lr。 |
| `get_current_learning_rate()` | 返回第一个 optimizer 的所有参数组 lr。 |

当前 `setup_schedulers()` 会 `pop('type')`，会修改配置字典。后续如果多次初始化或 resume 时遇到 scheduler 配置丢失，建议改为先 `deepcopy(train_opt['scheduler'])`。

### 保存与恢复

| 函数 | 作用 |
| --- | --- |
| `save_network(net, net_label, current_iter)` | 单独保存网络权重到 `opt['path']['weights']`。 |
| `load_network(net, load_path, strict=True)` | 从权重文件加载网络参数。 |
| `_print_different_keys_loading(...)` | 加载前打印缺失、多余、尺寸不匹配的 key。 |
| `save_checkpoint(epoch, current_iter)` | 保存训练恢复所需状态到 `opt['path']['checkpoints']`。 |
| `resume_checkpoint(resume_state)` | 从 checkpoint 恢复 `net_g/net_g_ema/optimizers/schedulers`。 |

当前 checkpoint 结构：

```python
{
    "epoch": epoch,
    "iter": current_iter,
    "net_g": self.get_bare_model(self.net_g).state_dict(),
    "net_g_ema": self.net_g_ema.module.state_dict() if self.net_g_ema else None,
    "optimizers": [...],
    "schedulers": [...],
}
```

这个结构对单生成器 SR 模型很直接，但对扩散模型可能不够通用。扩散模型可能需要额外保存：

- `diffusion` 配置，例如 beta schedule、采样步数、参数化方式。
- 多个网络，例如 `denoiser`、`condition_encoder`、`vae`。
- AMP scaler，例如 `GradScaler`。
- 随机状态，如需严格可复现。

### loss 汇总

| 函数 | 作用 |
| --- | --- |
| `reduce_loss_dict(loss_dict)` | 分布式训练时对 loss 做 reduce 平均，返回 Python float 日志字典。 |

子类的 `optimize_parameters()` 应把参与日志的 tensor loss 放入 `OrderedDict`，再调用该函数：

```python
self.log_dict = self.reduce_loss_dict(loss_dict)
```

## 4. SRModel 成员和函数说明

`SRModel` 是当前确定性单图像超分模型实现。

### 主要成员

| 成员 | 作用 |
| --- | --- |
| `self.net_g` | 生成器网络，例如 SwinIR。 |
| `self.load_path` | 预训练生成器权重路径。 |
| `self.net_g_ema` | EMA 生成器。仅当 `train.ema_decay > 0` 时创建。 |
| `self.cri_pix` | 像素损失，例如 L1/MSE。 |
| `self.cri_perceptual` | 感知/风格损失。 |
| `self.optimizer_g` | 生成器优化器。 |
| `self.lq` | 当前 batch 的低质量输入。 |
| `self.gt` | 当前 batch 的 GT。 |
| `self.output` | 当前模型输出。 |
| `self.metric_results` | 验证指标累计值。 |

### 构建与训练

| 函数 | 作用 |
| --- | --- |
| `__init__(opt)` | 构建 `net_g`，移动到 device，打印网络，加载预训练权重，训练时初始化训练组件。 |
| `init_training_settings()` | 设置 train 模式，初始化 EMA、loss、optimizer、scheduler。 |
| `init_losses()` | 根据 `train.pixel_opt` 和 `train.perceptual_opt` 构建 loss。 |
| `setup_optimizers()` | 收集 `net_g` 中 `requires_grad=True` 的参数，创建 `optimizer_g` 并加入 `self.optimizers`。 |
| `feed_data(data)` | 读取 `data['lq']` 和可选 `data['gt']` 到 device。 |
| `optimize_parameters(current_iter)` | 执行一次 SR 训练：`net_g(lq)`、计算 loss、反向、更新 optimizer、更新 EMA。 |

### 推理与验证

| 函数 | 作用 |
| --- | --- |
| `test()` | 使用 EMA 网络或 `net_g` 推理，输出到 `self.output`。 |
| `test_selfensemble()` | 测试时自集成：8 种翻转/转置增强分别推理，反变换后平均。 |
| `dist_validation(...)` | 分布式验证入口，仅 rank 0 执行非分布式验证。 |
| `nondist_validation(...)` | 遍历验证集，推理、转图像、保存图像、计算指标、记录最佳指标。 |
| `_log_validation_metric_values(...)` | 输出验证指标日志，并可写入 wandb。 |
| `get_current_visuals()` | 返回 `lq/result/gt` 的 CPU tensor。 |

### 保存

| 函数 | 作用 |
| --- | --- |
| `save(epoch, current_iter)` | 保存 `net_g` 权重，并调用 `save_checkpoint()` 保存训练状态。 |

## 5. 子类最小实现契约

如果新增一个模型类继承 `BaseModel`，至少建议实现：

```python
class MyModel(BaseModel):
    def __init__(self, opt):
        super().__init__(opt)
        # build network(s)
        # move to device
        # load pretrained weights
        # if train: init losses / optimizers / schedulers

    def feed_data(self, data):
        ...

    def optimize_parameters(self, current_iter):
        ...

    def test(self):
        ...

    def get_current_visuals(self):
        ...

    def nondist_validation(self, dataloader, current_iter, wandb_logger, save_img):
        ...

    def save(self, epoch, current_iter):
        ...
```

如果参与训练，还应保证：

- 至少一个 optimizer 被加入 `self.optimizers`。
- 如果使用 scheduler，调用 `self.setup_schedulers()` 或自定义调度器并加入 `self.schedulers`。
- `self.log_dict` 在 `optimize_parameters()` 后可被 `get_current_log()` 读取。
- 如果想复用当前 checkpoint，保持主网络名为 `self.net_g`。

## 6. 扩散模型超分的设计建议

你计划拆成两类：

1. 扩散模型框架类
2. 扩散模型框架用于超分类

这个方向是合理的。建议进一步明确两层边界：

```text
BaseModel
  ├── SRModel                     # 确定性超分，如 SwinIR
  └── DiffusionSRModel            # 训练/验证/保存层，面向超分任务

DiffusionProcess / GaussianDiffusion
  └── 负责 q_sample、p_losses、p_sample_loop、DDIM/DDPM sampler

net_g / denoiser
  └── 具体神经网络，如 UNet、SwinIR-like denoiser、条件编码器
```

### 6.1 扩散模型框架类建议

建议创建一个不继承 `BaseModel` 的纯算法类，例如：

```python
class GaussianDiffusion:
    def __init__(self, denoise_fn, betas, objective='pred_noise', ...):
        self.denoise_fn = denoise_fn
        ...

    def q_sample(self, x_start, t, noise=None):
        ...

    def p_losses(self, x_start, cond, t, noise=None):
        ...

    @torch.no_grad()
    def sample(self, cond, shape, steps=None):
        ...
```

职责边界：

| 职责 | 是否放进扩散框架类 |
| --- | --- |
| beta/alpha schedule | 是 |
| timestep 采样 | 是 |
| 前向加噪 `q_sample` | 是 |
| 训练目标 `pred_noise/pred_x0/v` | 是 |
| DDPM/DDIM/PLMS 采样 | 是 |
| loss 内部计算 | 可以是 |
| dataloader 数据移动 | 否 |
| optimizer/scheduler | 否 |
| checkpoint 保存 | 否 |
| validation 图像保存/指标 | 否 |

这样做的好处是扩散过程可复用：同一套 `GaussianDiffusion` 可以服务于 SR、去噪、修复等任务。

### 6.2 扩散模型用于超分类建议

建议新增：

```python
class DiffusionSRModel(BaseModel):
    ...
```

该类负责把 SimpleSR 训练框架和扩散算法粘起来：

| 函数 | 建议职责 |
| --- | --- |
| `__init__` | 构建 denoiser 网络 `net_g`，构建 `GaussianDiffusion`，加载预训练。 |
| `init_training_settings` | 初始化 EMA、loss 或 diffusion loss 权重、optimizer、scheduler。 |
| `feed_data` | 保存 `lq/gt`，必要时构建条件输入。 |
| `optimize_parameters` | 从 `gt` 采样 timestep，调用 `diffusion.p_losses(gt, cond=lq, t=t)`，反向更新。 |
| `test` | 使用 `diffusion.sample(cond=lq)` 逐步采样得到 SR 图。 |
| `nondist_validation` | 可复用 `SRModel` 的验证逻辑，但需要注意扩散采样很慢。 |
| `save` | 保存 `net_g`、EMA、checkpoint，以及扩散采样配置。 |

训练伪代码：

```python
def optimize_parameters(self, current_iter):
    self.optimizer_g.zero_grad()

    loss, log_dict = self.diffusion.p_losses(
        x_start=self.gt,
        cond=self.lq,
    )

    loss.backward()
    self.optimizer_g.step()

    self.log_dict = self.reduce_loss_dict(log_dict)

    if self.ema_decay > 0:
        self.update_ema(self.net_g, self.net_g_ema)
```

推理伪代码：

```python
@torch.no_grad()
def test(self):
    denoiser = self.get_ema_module(self.net_g_ema) if hasattr(self, 'net_g_ema') else self.net_g
    denoiser.eval()
    self.output = self.diffusion.sample(
        denoise_fn=denoiser,
        cond=self.lq,
        shape=(self.lq.size(0), 3, self.lq.size(2) * self.scale, self.lq.size(3) * self.scale),
    )
```

### 6.3 条件超分输入设计

扩散 SR 常见条件输入有几种：

| 条件方式 | 说明 |
| --- | --- |
| 直接拼接 | 将 noisy HR 与上采样 LR 在 channel 维拼接。简单直接。 |
| 条件编码器 | 单独用 encoder 提取 LR features，在 UNet 中 cross-attention 或 feature injection。 |
| latent diffusion | LR/HR 先进入 VAE latent，扩散发生在 latent 空间。复杂但省显存。 |
| residual diffusion | 预测 HR 或预测 residual `HR - upsample(LR)`。 |

建议第一版采用最简单的直接拼接：

```text
denoiser input = concat(x_t, upsample(lq), dim=1)
target = noise 或 x0
```

这样更容易接入现有数据集和验证逻辑。

### 6.4 类名与文件组织建议

建议文件结构：

```text
simplesr/
  models/
    base_model.py
    base_sr_model.py
    diffusion_sr_model.py
  diffusion/
    gaussian_diffusion.py
    schedules.py
    samplers.py
  networks/
    unet_arch.py
    diffusion_modules.py
```

也可以先不新建 `diffusion/` 包，把核心写到 `models/diffusion_sr_model.py` 中；但从长期维护看，扩散过程最好独立出来。

### 6.5 checkpoint 建议

当前 `BaseModel.save_checkpoint()` 假设主网络叫 `net_g`。扩散 SR 如果仍使用 `net_g` 表示 denoiser，可以直接复用。

建议后续扩展 checkpoint：

```python
state = {
    'epoch': epoch,
    'iter': current_iter,
    'net_g': self.get_bare_model(self.net_g).state_dict(),
    'net_g_ema': self.net_g_ema.module.state_dict() if self.net_g_ema else None,
    'optimizers': [...],
    'schedulers': [...],
    'diffusion': {
        'schedule': self.diffusion.schedule_name,
        'num_timesteps': self.diffusion.num_timesteps,
        'objective': self.diffusion.objective,
    }
}
```

如果未来有多个网络，不建议继续把所有模型硬编码进 `BaseModel.save_checkpoint()`；可以让子类覆盖 `save_checkpoint()` 或新增更通用的 `get_checkpoint_networks()`。

### 6.6 验证策略建议

扩散模型验证比确定性模型慢很多，不建议每次验证都跑完整采样。建议支持：

- `val.sample_steps`: 验证采样步数，例如 20/50。
- `val.sample_method`: `ddpm` / `ddim`。
- `val.max_images`: debug 或快速验证时限制图像数量。
- `val.save_img`: 只在需要时保存图像。

配置示例：

```yaml
model:
  path: simplesr.models.diffusion_sr_model.DiffusionSRModel

network_g:
  path: simplesr.networks.unet_arch.UNetDenoiser
  kwargs:
    in_channels: 6
    out_channels: 3
    base_channels: 64

diffusion:
  timesteps: 1000
  beta_schedule: linear
  objective: pred_noise
  sample_method: ddim
  sample_steps: 50

train:
  ema_decay: 0.999
  optim_g:
    type: AdamW
    lr: 1.0e-4
    weight_decay: 0
```

## 7. 建议优先级

建议按以下顺序推进：

1. 先稳定 `BaseModel` 的通用接口，尤其是 checkpoint 对单网络/多网络的边界。
2. 新增 `GaussianDiffusion`，先用随机 tensor 做单元级 shape 测试。
3. 新增一个最小 `UNetDenoiser` 或复用已有网络作为 denoiser。
4. 新增 `DiffusionSRModel`，先完成训练 loss 前向和日志。
5. 再接入采样、验证、保存图像和指标。
6. 最后考虑 self-ensemble、DDIM 加速、latent diffusion 等增强功能。

第一版目标应是“能训练、能 resume、能采样验证”，不要一开始就把多采样器、多条件注入、多网络 checkpoint 全部做复杂。
